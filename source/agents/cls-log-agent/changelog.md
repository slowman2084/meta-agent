## 2026-03-11 prompt.md 优化（基于评估诊断）

**触发原因**：test_20260310_1134 评估发现 5 个未达标用例（平均分 89.34），诊断出3类真实 prompt 短板。

**修改内容**：

1. **采样规范强化**（异常处理表格，第五步）
   - 原：`时间范围 >24h | 摘要中输出采样警告，启用采样`
   - 新：拆分为两档：≥48h 必须 `SamplingRate=0.1`；>24h 且 <48h 视超时降级，摘要注明时间跨度
   - 修复用例：Case 5（7天未采样）、Case 7（38小时未采样）、Case 23（7天未采样）

2. **统计输出完整性**（输出规范类型B模板 + 红线新增第9条）
   - 类型B表格模板新增"占比"列，体现绝对数量+百分比并列格式
   - 新增红线9：禁止统计输出只有百分比，必须同时呈现绝对数量和百分比
   - 修复用例：Case 9（仅输出百分比缺绝对数量）、Case 21（高危占比未计算）

3. **安全场景建议约束**（红线新增第11条）
   - 新增红线11：涉及攻击/拦截/告警/异常访问等安全场景，必须包含至少一条可操作后续建议
   - 修复用例：Case 21（安全告警无后续建议）

**同步状态**：已通过 `scripts/install.py` 同步至 Cursor / CodeBuddy / Claude Code / Codex 四个目标。
