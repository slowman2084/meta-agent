# meta-debug 理想态

## 角色定位

meta-debug 是评估体系初始化阶段的调试工具，不是优化工具。它的价值在于在进入 evo_looper 之前，发现三元组（提示词 / 理想态 / rubrics）自身的设计缺陷，避免后续迭代在错误的评估基准上空转。

## 输出质量标准

### 1. 诊断有据可查

每条诊断都必须有引用原文的证据。没有证据的诊断不输出——宁可漏报，不要误报。

### 2. 四类问题覆盖完整

| 问题类型 | 描述 |
|---------|------|
| Rubric 自身设计问题 | criterion 过严/过松/模糊/权重失衡 |
| Rubric ↔ 理想态矛盾 | rubric 要求与理想态描述方向不一致，或理想态有要求但 rubric 无对应 criterion |
| 理想态 ↔ 提示词矛盾 | 提示词遗漏了理想态要求，或包含与理想态相悖的指令 |
| 用户价值视角洞察 | 从 ≥3 条实际输出中发现值得质疑当前设计的模式，以疑问形式呈现 |

### 3. 决策项设计合理

- 每个决策项对应一个独立的修改决策
- 每个决策项必须包含 skip 选项
- 选项的 action_type 与实际可执行的操作一致
- priority:1 不滥用，只用于高严重度的 rubric / ideal_state 问题

### 4. JSON 格式严格符合接口规范

`calibration_report.json` 的字段名、枚举值、数组类型与 `calibration_viewer.html` 期望的接口完全一致，加载后无解析错误。

### 5. 克制的用户价值洞察

用户价值洞察以疑问形式呈现，不替用户做判断。必须有 ≥3 条用例的实证支撑，避免基于个案的过度推断。

## 典型的不合格输出

- 诊断措辞模糊，无具体引用（如「rubric 描述不够清晰」，但不引用具体 criterion）
- 用户价值洞察基于单条用例（如「Case 3 的输出似乎更有用」）
- decisions 中缺少 skip 选项
- JSON 字段名与接口规范不符（如用 `case_ids` 代替 `affected_cases`）
- 把 `category: "prompt"` 的诊断填写了 `subcategory`
